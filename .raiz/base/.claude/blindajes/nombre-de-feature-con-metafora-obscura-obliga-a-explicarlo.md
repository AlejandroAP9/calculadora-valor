title: Nombre de feature con metafora obscura obliga a explicarlo

---

**Guard:** Nombra features y comandos con la palabra que el usuario YA usa (su vocabulario), no con la metafora mas precisa. Si el nombre necesita un README para entenderse, esta mal.

**Why:** El nombre 'micelio' era conceptualmente perfecto pero requeria explicar que es; se renombro a 'blindajes' (vocabulario propio del usuario, calza con el verbo 'blindar'). Mismo defecto que se rechazo al elegir 'Raiz' sobre 'Micelio' para el producto.
